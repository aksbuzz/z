PyEdit is a new editor, for forgotten langages essentially.

Requirements

I've actually not tested other versions, so please report me bugs.

  • Python > 2.5.1 ;
  • PyQt 4.3.1 ;
  • DejaVu Sans Mono fonts [optionnal, but very recommended];

Features

With each of these file types, it is possible to compile them, view the output in different formats. In fact, you just have to load the file and follow the Commands menu/shortcuts to view all possible actions. Actions are configurable via an XML file format.

For example, this site is just a file and I just ask PyEdit to produce HTML output from it via the F1 key. I can then view the output in Firefox via F2.

Here's a non limited list of supported langages :

  • reStructuredText ;
  • Python ;
  • Asymptote ;
  • LaTeX ;
  • Other formats needs support at the moment;

Plugins

PyEdit supports a limited plugins architecture. At the time writing this page, here are the implemented plugins :

  • TextMate-like snippets ;
  • Multicursors editing and a video ;
  • SmartIndent;

In the Scheme directory you'll find XML files. Each one contains :

  • the grammar and syntax colorisation (you can add your own keywords);
  • the snippets definitions for a given langage;

You can edit them to add/sub whatever you want.

Commands

From PyEdit, you can also define your own commands. They're inside the commands directory: again an XML file. Look at the given samples to give you an idea about what's going on.

Limitations

  • Only use UTF-8 encoding for the moment;
  • Syntax Highlighting maybe very slow with large documents;

Performances

PyEdit is pretty slow, that's a fact because I choose not to use the Scintilla control (I don't like the API behind it). So it needs to do all the drawings by itself. If you need more performance, toogle-off some options in the View menu, particulary :

  • the line numbers margin ;
  • the white spaces/tabs;
